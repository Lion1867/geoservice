# ГеоНейро (GeoAI) — Интеллектуальная система поддержки принятия решения по градостроительному анализу территорий

> Интеллектуальная система для анализа градостроительной документации с применением технологий Retrieval-Augmented Generation (RAG) и YandexGPT. Система отвечает на вопросы пользователей, опираясь на актуальные нормативно-правовые акты РФ в области строительства и землепользования.

## Обзор архитектуры

Система построена по микросервисной архитектуре и включает пять ключевых компонентов:

```
┌─────────────────────────────────────────────────────────┐
│                    Клиент (Frontend)                     │
└───────────────────────────┬─────────────────────────────┘
                            │ HTTP
┌───────────────────────────▼─────────────────────────────┐
│              Основной бэкенд (main.py)                   │
│   Оркестратор: гео-аналитика + кадастр + RAG-запросы    │
└────────────┬──────────────────────────────┬─────────────┘
             │ HTTP                         │ HTTP
┌────────────▼──────────────┐  ┌────────────▼─────────────┐
│    Гео/кадастровые        │  │   RAG-микросервис         │
│    внешние API            │  │   (rag_server.py)         │
│    (Яндекс Карты и др.)   │  │   FastAPI + Qdrant        │
└───────────────────────────┘  └────────────┬─────────────┘
                                            │
                               ┌────────────▼─────────────┐
                               │    Yandex Embedder        │
                               │    + Qdrant (векторная БД)│
                               └────────────┬─────────────┘
                                            │
                               ┌────────────▼─────────────┐
                               │    YandexGPT              │
                               │    (генерация ответа)     │
                               └──────────────────────────┘
```

**Поток обработки запроса:**

1. Пользователь отправляет вопрос через интерфейс.
2. Основной бэкенд параллельно собирает данные через гео-аналитический и кадастровый модули.
3. Если вопрос касается строительных норм — запрос перенаправляется в RAG-микросервис.
4. RAG-микросервис через Yandex Embedder создаёт вектор запроса и выполняет семантический поиск в Qdrant.
5. Найденные нормативные фрагменты возвращаются оркестратору.
6. Вся собранная информация передаётся в YandexGPT для генерации финального ответа.
7. Ответ возвращается пользователю на клиентскую часть.

## Микросервисы

Система запускается двумя независимыми процессами. Каждый — отдельный FastAPI-сервер на своём порту.

---

### Основной бэкенд — `backend/main.py`

**Роль:** Оркестратор. Принимает запросы от фронтенда, агрегирует данные из геомодулей и RAG-сервиса, передаёт итоговый контекст в YandexGPT и возвращает ответ.

**Порт по умолчанию:** `8000`

**Основные функции:**

- Приём пользовательских запросов через REST API.
- Параллельный сбор геоаналитической и кадастровой информации.
- Маршрутизация нормативных вопросов в RAG-микросервис (порт `8003`).
- Агрегация всех данных и формирование промпта для YandexGPT.
- Возврат структурированного ответа с источниками.

**Запуск:**

```bash
cd backend
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

---

### RAG-микросервис — `backend/rag_core_final/rag_server.py`

**Роль:** Специализированный сервис семантического поиска по нормативно-правовой базе знаний. Принимает текстовый запрос, ищет релевантные фрагменты в Qdrant и возвращает топ-N результатов.

**Порт по умолчанию:** `8003`

**Пайплайн работы:**

```
Запрос (текст)
    │
    ▼
Yandex Embedder
(текст → вектор 1536 dims)
    │
    ▼
Qdrant — косинусный поиск
(топ-5 ближайших чанков)
    │
    ▼
Возврат фрагментов + метаданных
(источник, номер статьи, релевантность)
```

**Параметры чанкинга:**

| Параметр | Значение |
|---|---|
| Метод | `RecursiveCharacterTextSplitter` (LangChain) |
| Размер чанка | 500 токенов |
| Перекрытие (overlap) | 10% (50 токенов) |
| Размерность вектора | 1 536 |

**Запуск:**

```bash
cd backend/rag_core_final
uvicorn rag_server:app --host 0.0.0.0 --port 8003 --reload
```

## База знаний (датасет)

RAG-микросервис работает на корпусе нормативно-правовых актов РФ в области градостроительства.

### Состав документов

| # | Документ |
|---|---|
| 1 | Градостроительный кодекс РФ |
| 2 | Земельный кодекс РФ |
| 3 | СНиП (актуализированные редакции — Своды правил, СП) |
| 4 | СанПиН — санитарные правила и нормы |
| 5 | ГОСТ в области строительства |
| 6 | Правила землепользования и застройки (ПЗЗ) |
| 7 | СП 4.13130 (Системы противопожарной защиты) и др. |

### Источники

- [pravo.gov.ru](https://pravo.gov.ru) — Официальный интернет-портал правовой информации
- [docs.cntd.ru](https://docs.cntd.ru) — Электронный фонд правовых и нормативно-технических документов
- [minstroyrf.gov.ru](https://minstroyrf.gov.ru) — Сайт Минстроя России

### Статистика датасета

| Показатель | Значение |
|---|---|
| Общий объём | ~10 МБ / ~3 000 000 символов |
| Количество документов | ~15 ключевых нормативных актов |
| Всего предложений | 14 000+ |
| Всего слов | ~302 000 |
| Уникальных слов | 26 837 |
| TTR (лексическое разнообразие) | 0.0889 |
| Средняя длина слова | 6.90 символов |
| Средняя длина предложения | 21.58 слов |
| Размерность эмбеддинга | 1 536 |

### Подготовка данных

Документы обрабатываются в несколько этапов:

```
PDF-документы
    │
    ▼  pdfplumber / pypdf
Извлечение текстового слоя (TXT)
    │
    ▼  NLTK
NLP-предобработка:
  - токенизация
  - лемматизация
  - удаление стоп-слов
    │
    ▼  LangChain RecursiveCharacterTextSplitter
Чанкинг (500 токенов, overlap 10%)
    │
    ▼  Yandex Embedder
Векторизация каждого чанка (1536 dims)
    │
    ▼  Qdrant
Сохранение в векторную базу данных
```

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/Lion1867/geoservice.git
cd geoservice
```

### 2. Создание виртуального окружения

```bash
python -m venv venv

# Linux / macOS
source venv/bin/activate

# Windows
venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install -r backend/requirements.txt
```

### 4. Настройка переменных окружения

```bash
mkdir backend/.env
# Заполните backend/.env своими ключами (см. раздел ниже)
```

### 5. Запуск микросервисов

Оба микросервиса запускаются **в отдельных терминалах**:

**Терминал 1 — RAG-сервис (запускать первым):**

```bash
cd backend/rag_core_final
uvicorn rag_server:app --host 0.0.0.0 --port 8003 --reload
```

**Терминал 2 — Основной бэкенд:**

```bash
cd backend
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

## Фронтенд

Клиентская часть представляет собой веб-интерфейс на HTML/CSS, взаимодействующий с основным бэкендом по REST-протоколу.

**Возможности:**

- Чат-интерфейс для общения с системой.
- Отображение источников нормативных документов под каждым ответом.
- Интерактивная карта (Яндекс Карты) для выбора геолокации.
- История диалога в рамках сессии.

## Этические и правовые аспекты

Все нормативные документы, используемые в базе знаний, являются официальными документами государственных органов РФ.

Система является **инструментом поддержки принятия решений**. Результаты анализа носят информационно-справочный характер и **не заменяют официальной градостроительной экспертизы** или консультации сертифицированного юриста.

Архитектура RAG обеспечивает привязку ответов к конкретным пунктам нормативных документов, что минимизирует риск «галлюцинаций» большой языковой модели.
